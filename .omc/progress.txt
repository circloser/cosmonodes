== 결정/제약 ==
- Supabase/Google OAuth는 사용자 제공 크리덴셜 필요 → 백엔드 연동은 follow-up(docs/SUPABASE_SETUP.md).
- 이번 런: 크리덴셜 없이 동작하는 프론트엔드 + LocalStorageDataProvider로 전체 핵심 루프 구현.
- 데이터 계층은 DataProvider 인터페이스로 분리 → SupabaseDataProvider로 교체 가능.
- 환경: Node 24, React 19, Vite 8, TypeScript 6, Tailwind 3, Vitest 4. tsconfig가 verbatimModuleSyntax/erasableSyntaxOnly/noUnusedLocals 강제 → import type 사용, enum 금지.

- src/domain/types.ts — 도메인 모델. GraphNode.note는 degree<=1만 존재(AC17 불변식).
- src/data/DataProvider.ts — 인터페이스(백엔드 교체 지점).
- src/data/LocalStorageDataProvider.ts — 저장 구현 + getGraph()가 degree-2에 note 미포함.
- src/data/seed.ts — 시드 + makeMatchedUserNetwork + makePerfDataset(300).
- src/data/DataProvider.test.ts — note 미노출/동의 게이트 단위테스트.
- src/store/useGraphStore.ts — Zustand 상태/액션 + perfMode.
- src/components/* — CosmicBackground, GraphView(react-force-graph-2d), NodeCard, AddStarModal(동의게이트+supernova), Supernova, ProfilePanel, FpsOverlay, AppShell.
- src/lib/colors.ts, src/lib/id.ts.
- README.md, docs/SUPABASE_SETUP.md.

- vitest/jsdom 타입이 프로덕션 tsc 빌드로 새지 않게: 테스트 설정은 vitest.config.ts(별도, tsc include 밖)로, tsconfig.app.json에서 *.test.ts/src/test exclude.
- 초기 `npm install --silent`가 dev deps 설치를 조용히 실패한 적 있음 → 비-silent 재설치로 복구.
